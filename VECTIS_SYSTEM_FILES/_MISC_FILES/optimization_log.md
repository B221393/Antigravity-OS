## 2026-01-13: UnifiedLLM Robustness Fixes

**Target:** `VECTIS_SYSTEM_FILES/modules/unified_llm.py`

### 1. Lazy Initialization of Gemini Client

- **Problem:** `UnifiedLLM` was attempting to initialize `GenerativeModel` immediately upon instantiation if `provider="gemini"`. If the library was missing or the API key invalid at startup, this caused a crash or `AttributeError` later.
- **Fix:** Moved initialization inside the `generate` method.
- **Mechanism:** Checks `getattr(self, 'client', None)` before usage. If missing, attempts to initialize.

### 2. Dynamic Model Switching

- **Problem:** Changing `self.model_name` (e.g., from "gemini-pro" to "gemini-flash") did not update the underlying `client` object if it was already created.
- **Fix:** Added a check in `generate` to compare `self.client.model_name` with `self.model_name`. If they differ, `self.client` is forced to `None` to trigger re-initialization.

### Implemented Code (Snippet)

```python
        if active_provider == "gemini":
            # Check for model switch
            if getattr(self, 'client', None):
                 current_client_model = getattr(self.client, 'model_name', None)
                 if current_client_model and current_client_model != self.model_name:
                     self.client = None # Force re-init

            # Lazy Init / Re-init checks
            if not getattr(self, 'client', None):
                if _HAS_GEMINI:
                    try:
                        self.client = GenerativeModel(self.model_name)
                    except Exception as e:
                        return f"[Gemini Init Error] {str(e)}"
                else:
                    return "[Gemini Error] Client library not imported."
```
