# VECTIS SYSTEM LAUNCHERS MANUAL

**最終更新日**: 2026-01-10

VECTIS OSの各アプリケーションを起動・制御するためのバッチファイル（`.bat`）の仕様書です。
これらのファイルは、システムのルートディレクトリに配置されています。

## 📂 保存場所 (Location)

すべての主要な起動スクリプトは以下のディレクトリにあります：

```
c:\Users\Yuto\OneDrive - Hiroshima University\app\VECTIS_SYSTEM_FILES\
```

---

## 🚀 メイン・ランチャー (推奨)

通常は、以下の番号付きファイルを使用してください。これらは最新の環境設定に合わせて整備されています。

### `01_Main_HUD.bat` (Command Center)

* **用途**: **メインのシステム操作画面 (Jarvis HUD)** を起動します。
* **形式**: Web UI (ブラウザが開きます)
* **URL**: `http://localhost:8515`
* **機能**: 他のアプリへのアクセス、システム状態の確認、AIエージェントとの対話など、VECTISのホーム画面です。

### `02_YouTube_CLI.bat` (Quick Digest)

* **用途**: **YouTube チャンネル要約 (高速版)** を起動します。
* **形式**: CLI (黒い画面のターミナル)
* **機能**:
  * キーボード操作のみでYouTubeの最新動画をチェック・要約。
  * 動作が非常に軽量。
  * Job HQへのデータ転送機能付き。

### `03_YouTube_Web.bat` (Visual Digest)

* **用途**: **YouTube チャンネル要約 (リッチ表示版)** を起動します。
* **形式**: Web UI (ブラウザが開きます)
* **URL**: `http://localhost:8519`
* **機能**:
  * サムネイル画像付きで動画一覧を表示。
  * リッチなUIで「圧縮/解凍」スタイルの要約を閲覧。
  * 音声読み上げ機能などに対応。

### `04_Auto_Watcher.bat` (Background Monitor)

* **用途**: **バックグラウンド監視システム** を起動します。
* **形式**: バックグラウンド (ターミナルが表示されます)
* **機能**:
  * クリップボード等の監視や、指定された自動タスク（URL検知・自動処理など）を実行し続けます。
  * 常駐させておくことで、コピー操作だけでVECTISの機能呼び出しなどが可能になります。

## 🛠️ その他のランチャー

特定の目的や、旧バージョンとの互換性のために残されているファイルです。基本的には上記の番号付き (`01`〜`04`) の使用を推奨しますが、用途に応じて使用可能です。

| ファイル名 | 説明 | 備考 |
| :--- | :--- | :--- |
| **`LAUNCH_EGO.bat`** | **Ego Evolution Engine** を単独で起動します。自我エンジンの思考プロセスを確認・デバッグする際に使用します。 | 開発者/デバッグ向け |
| **`LAUNCH_HUD.bat`** | デスクトップ常駐型 (軽量版) のHUDを起動します。Web版 (`01`) とは異なる古いGUI、またはウィジェット版です。 | 軽量動作が必要な場合 |
| **`LAUNCH_CHANNEL_CLI.bat`** | `02_YouTube_CLI.bat` と同等機能。 | エイリアス (別名) |
| **`LAUNCH_CHANNEL_DIGEST.bat`** | `03_YouTube_Web.bat` と同等機能。 | エイリアス (別名) |

## 📂 `LAUNCHERS` フォルダについて

`LAUNCHERS/` ディレクトリ内にあるファイル（`1_Start_HUD.bat` 等）は、バックアップまたは整理用です。
ルートディレクトリにあるファイルが最新かつ正しく設定されていますので、そちらを使用してください。

## ⚠️ トラブルシューティング

* **画面が一瞬で消える場合**: エラーが発生しています。`.bat` ファイルの最後の `pause` が機能していないか、Python環境 (`.venv`) に問題がある可能性があります。
* **モジュールが見つからない場合**:
    1. 起動時に `.venv\Scripts\activate.bat` が正しく呼ばれているか確認してください（すべての推奨バッチには記述済みです）。
    2. 必要なライブラリがインストールされているか確認してください。
