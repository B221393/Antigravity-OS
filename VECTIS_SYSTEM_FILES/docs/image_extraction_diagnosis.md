# 画像抽出問題の診断と解決策

## 問題の概要

`extract_notes_vision.py`を実行して画像からテキストを抽出しようとしましたが、**全ての画像でAPIレート制限(429エラー)により失敗**しました。

## 診断結果

### 1. **Gemini API無料枠の制限**

- **リクエスト数制限**: 約15リクエスト/分
- **1日制限**: 約1000リクエスト/日 (無料枠)
- **現在の状況**: おそらく無料枠を使い切った可能性が高い

### 2. **元のスクリプトの問題点**

- リクエスト間の待機時間が15秒と短すぎる (1分に4リクエスト = 制限超過)
- 429エラー時の待機時間は70秒あるが、次のリクエストで再び制限に引っかかる
- 失敗したエントリをファイルに保存してしまい、リトライ時に再処理されない

### 3. **処理対象画像**

合計 **23枚の画像ファイル**:

- JPGファイル: 12枚 (サイズ: 1.3MB〜3.5MB)
- PNGファイル: 11枚 (サイズ: 204KB〜1.4MB)

## 解決策

### 改善版スクリプト: `extract_notes_vision_v2.py`

#### 主な改善点

1. **より保守的なレート制限対応**
   - ベース待機時間: 15秒 → **30秒**
   - 429エラー時: 70秒 → **90秒**
   - これにより1分あたり約2リクエストで安全に実行

2. **失敗エントリの自動削除**
   - スクリプト起動時に失敗した画像を自動検出
   - 再実行時に自動的に再処理

3. **詳細な進捗表示**
   - 現在時刻、ファイルサイズ、成功/失敗状態を表示
   - APIクォータ消費の警告

4. **実験モデルの使用**
   - `gemini-2.0-flash` → `gemini-2.0-flash-exp`
   - 実験モデルの方がクォータが別枠の可能性

## 推奨アクション

### オプション A: 24時間待機 (無料で確実)

1. **24時間後**に無料枠がリセットされるのを待つ
2. `extract_notes_vision_v2.py`を実行
3. 約12分で完了 (23枚 × 30秒/枚)

### オプション B: 別のAPIキーを使用 (即時)

1. 新しいGoogleアカウントで別のGemini APIキーを取得
2. `.env`ファイルのキーを更新
3. すぐに実行可能

### オプション C: 有料プランにアップグレード (大規模処理向け)

- より高いレート制限 (数百リクエスト/分)
- より大きな1日制限

## 実行コマンド

```bash
# 改善版スクリプトを実行
python extract_notes_vision_v2.py
```

スクリプトは中断可能で、再実行時に自動的に続きから処理を開始します。

## 注意事項

- **APIクォータがリセットされるまで待つことを推奨**
- 無理に実行すると全て429エラーで失敗する可能性が高い
- 処理中は`extracted_notes_draft.md`に随時保存されます
