## 1. 優れたTextualアプリの詳細分析

### Harlequin (SQL IDE)

- **特徴**: ターミナル内でSQL実行、結果テーブル表示
- **学ぶ点**: 入力→結果のリアルタイム反映
- **VECTIS適用**: 知識グラフの検索・フィルタ機能

### Elia (ChatGPT Client)

- **特徴**: 会話履歴+入力欄+レスポンス表示
- **学ぶ点**: 3パネル構成（履歴|会話|入力）
- **VECTIS適用**: Gemini CLIを同様の3パネル化

### Toolong (Log Viewer)

- **特徴**: 複数ログファイルのリアルタイム監視
- **学ぶ点**: ログの自動スクロール、検索機能
- **VECTIS適用**: River Daemonのログを常時表示

### Memray (Memory Profiler)

- **特徴**: リアルタイムメモリグラフ
- **学ぶ点**: 数値→視覚化（プログレスバー、グラフ）
- **VECTIS適用**: 知識量のビジュアライゼーション

## 2. 現行VECTISの問題点（詳細）

### 2.1 設計思想の問題

| 現状 | 問題 | 本来あるべき姿 |
|------|------|----------------|
| 「ランチャー」として設計 | 別ウィンドウ前提の思想 | **ダッシュボード**として設計 |
| UI仕様書(v2.0)が古い | 2パネル構成だが実装と乖離 | v5.0に合わせて更新必要 |
| 13個のアプリ登録 | 多すぎて選択疲れ | **最重要の4-6個に絞る** |

### 2.2 削除すべき要素

#### 🔴 即削除推奨

| 項目 | 理由 |
|------|------|
| **Color Alchemy** | Streamlit外部起動、VECTISの本質と無関係 |
| **Ascii Artist** | 同上 |
| **UX Labs** | 未完成、動作しない可能性高い |
| **Intelligence Hub (旧TUI)** | vectis_nexus.pyに統合済み、重複 |

#### 🟡 統合推奨

| 項目 | 統合先 |
|------|--------|
| **River Flow + Daemon** | 1つの「情報収集」として統合 |
| **Deep Researcher + Interview Coach** | 「就活支援」として統合 |
| **3つのゲーム** | 「息抜き」として1メニューに |

### 2.3 残すべきコア機能

```
🧠 情報収集（自動で動く、状態表示のみ）
📊 知識グラフ（検索・閲覧）
💼 就活支援（企業分析・面接練習）
🎮 息抜き（ゲーム類を1ボタンに）
⚙️ 設定（収集間隔など）
```

## 6. 結論

**「機能を増やす」ではなく「本質に絞る」**

VECTISの目的は「就活情報の自動収集と活用」。

- Color Alchemy? 不要
- Ascii Artist? 不要
- 複雑なカテゴリ? 不要

**4つのボタンで十分。**
