# AI協同確認テスト：3D空間における光の散乱シミュレーション

## 目的
複数のAIモデルに同一プロンプト・同一条件でコードを書かせ、実装の違いや品質差を比較する

## 統一プロンプト

```
【課題】Three.jsを使用したリアルタイム3D光散乱シミュレーター

【要件】
1. Three.jsで3D空間を初期化（カメラ、ライト、レンダラー）
2. パーティクルシステムで光の散乱を表現
   - 光源から放出される複数のビーム/粒子
   - 各粒子が球体などのオブジェクトと衝突
   - 衝突後の散乱方向はランダム（Lambertian reflectionを簡易実装）
3. パラメータ：
   - 初期光子数: 1000
   - 物体の反射率: 0.7
   - シミュレーション時間: 無制限（リアルタイム）
4. ビジュアル：
   - 粒子軌跡を線で表示
   - 衝突箇所をドットで表示
   - カラーマップで衝突回数を可視化（青→緑→赤）
5. UI：
   - 光源方向調整（スライダー）
   - 粒子数増減（ボタン）
   - シミュレーション一時停止・再開
   - フレームレート表示

【出力形式】
- HTMLファイル（1ファイル: Three.jsスクリプト含む）
- 動作確認可能な完全なコード
- 250行以上（ただし冗長さは避ける）

【条件】
- 依存関係：Three.js v0.128.0（CDNから）
- ターゲット: モダンブラウザ（Chrome, Firefox, Safari）
- パフォーマンス: 1000粒子で60FPS維持を目指す
```

## 評価軸
- **機能完成度**: 要件充足度
- **コード品質**: 可読性、保守性、効率性
- **パフォーマンス**: FPS、粒子処理速度
- **実装アプローチ**: アルゴリズム選択の違い
- **ユニークな工夫**: 各AIならではの実装

## 結果記録

### AI #1: Claude Haiku (light_scatter_ai1_haiku.html)
**アプローチ**: 単純な距離ベース衝突検出
- **強み**: シンプルで理解しやすい、高速初期実装
- **弱み**: 物理学的正確さが低い
- **実装時間**: 高速
- **パフォーマンス**: 1000粒子で60FPS
- **ビジュアル**: 青→赤色変化
- **UI機能**: ライト方向調整、一時停止、リセット

### AI #2: Physics-Optimized (light_scatter_ai2_optimized.html)
**アプローチ**: クラス設計、反射損失計算
- **強み**: より正確な物理演算、メモリ効率的、スケーラブル
- **弱み**: 複雑なコード構造
- **特徴**: PhotonSystemクラス、動的サイズ変更対応
- **パフォーマンス**: 1500粒子で60FPS
- **ビジュアル**: 色相循環、エネルギーメーター
- **UI機能**: 粒子数動的変更、反射損失調整、スケーラビリティ

### AI #3: Raycasting (light_scatter_ai3_raycasting.html)
**アプローチ**: 本格的なレイトレーシング + 幾何学選択
- **強み**: 物理的に正確（反射の法則）、複数ジオメトリ対応
- **弱み**: 計算量が多い
- **特徴**: THREE.Raycaster使用、複数形状（球・立方体・トーラス）、経路長追跡
- **パフォーマンス**: 2000レイで60FPS
- **ビジュアル**: バウンス数に基づく色分け
- **UI機能**: レイ数調整、バウンス数制限、ジオメトリ切り替え
