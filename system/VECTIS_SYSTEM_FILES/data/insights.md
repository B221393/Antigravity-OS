# VECTIS Knowledge Base - Insights

## 1. 技術力の定義（2026-02-02）

**出典**: YouTube動画「技術力の高いエンジニアとは？」（IT技術屋のボヤキ）

### 核心的洞察

#### 技術力はベクトル量である

- 技術力 $T$ は単なる知識量（スカラー）ではない
- `Value = T⃗ · P⃗`（技術ベクトル × 問題解決ベクトル）
- 課題と直交していれば、いくら技術があっても仕事量はゼロ

#### エントロピー抑制の重要性

- 複雑なコード = 高エントロピー = 保守コスト増大
- 真の技術力 = シンプルで誰にでも読めるコードを書く能力

#### 成長の運動方程式

- 基礎（F=ma）を先に理解せよ：データ構造、ネットワーク、OS
- アクション：ソースコードを読む、アウトプットする、エラーを歓迎する

### 戦略的示唆（ハイブリッド戦略）

- **剣（短期）**: 流行りの技術で採用フィルターを通過
- **盾（長期）**: 基礎力で現場サバイバル

## 2. Claude公式 SKILL.md ガイド（2026-02-02）

**出典**: Anthropic公式ドキュメント解説動画

### 核心的洞察

#### SKILL.mdとは

- `.claud/skills/` に配置するプロンプト仕様書
- YAML Front Matter（メタデータ）+ Markdown（本文）
- LLMに対する「実験プロトコル」のインストール

#### 構造

```yaml
---
name: スキル名
description: いつ使うか（Trigger）
---
具体的な手順書（Markdown）
```

#### 利点

1. **再現性向上**: 毎回同じ品質のアウトプット
2. **エントロピー抑制**: 必要な時だけロード（遅延評価）
3. **ポータビリティ**: Git管理でチーム共有可能

### 戦略的示唆

- プロンプトエンジニアリング → AIオーケストレーションへの進化
- VECTISにも `skills/` ディレクトリを標準化すべき
