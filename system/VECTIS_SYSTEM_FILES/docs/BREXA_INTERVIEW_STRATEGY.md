## 🎯 面談の目的
給与・待遇面での懸念を解消し、**「この会社が自分の人生と技術投資を預けるに値するか」**を冷徹に判断する。納得できない場合は「内定辞退」を選択肢に入れる。

## 💰 【最重要】給与・待遇に関する直接的質問
アウトソーシング業界のプロに対し、他社比較を交えて本音を引き出します。

1. **市場価値と単価の連動性**
   - 「私の現在の技術スタック（自律システム構築、NVIDIA Warp研究等）を市場に投入した際、クライアントからの『提示単価（チャージ）』は他社と比較してどう設定されますか？また、その単価はどの程度自身の給与に反映されますか？」
2. **昇給カーブの透明性**
   - 「入社後、スキル向上に伴う昇給の『具体的な基準と実績』を教えてください。メイテック等の大手と比較して、若手の給与の伸び率にどのような優位性がありますか？」
3. **実質的な可処分所得（福利厚生）**
   - 「住宅補助、寮制度、手当等を含めた『手取り額のリアル』について、同じエリア（勤務地）の他社事例と比較してどうですか？額面以上に差が出るポイントを伺いたいです。」
4. **待機時のリスク**
   - 「万が一、プロジェクト間の待機期間が発生した場合、給与や賞与にどのような影響がありますか？精鋭集団である貴社において、そのリスクをどうコントロールしていますか？」

## 🛠️ 技術とキャリアに関する戦略的質問
「ただの派遣」で終わらないための確認です。

1. **プロジェクトの「質」の決定的な差**
   - 「テクノプロやメイテックを知る〇〇様から見て、貴社が『あえて小規模・精鋭であること』で獲得できている、他社では経験できない『上流かつ先端』なプロジェクトの具体例を教えてください。」
2. **自律型エンジニアの評価**
   - 「私は自身の開発環境をAIで自動化するシステム（Antigravity）を構築しています。このような『仕組みを創り、個の出力を最大化する姿勢』は、貴社内でどのように評価・活用されますか？」

- [ ] 福利厚生（住宅等）を含めた実質賃金が、自身の期待値を超えているか？
- [ ] 担当者が「あなたの技術的野心（Antigravity等）」を正しく理解し、高評価しているか？
- [ ] 提示されたキャリアパスが、将来の「巨大インフラ企業（NYK, MHI等）」へのステップアップに寄与するか？

### 1. 給与・待遇の現実解
- **モデル年収**: 40歳時 約900万円。
    - 超大手メーカーには一歩譲るが、同業界内ではトップクラス。
    - 初任給を無理に上げるのではなく、スキルと連動した昇給ロジック。
- **透明性**: 株主配当に偏重せず（アルプス技研との対比）、社員還元と内部留保のバランスを重視。

### 2. 勤務地とライフスタイルの「権利」
- **姫路配属の確約**: 
    - メイテックや他社が「最大限考慮（＝一切の約束なし）」に留まる中、内定段階での「姫路支店配属」を確約。
    - 実家から5分以内での勤務、あるいは地元拠点での生活という**「安心感の提供」を会社の努力で保証**している点は非常に珍しい。

### 3. キャリアの自律性
- **技術の選択権**: 
    - 6ヶ月ごとの意向確認、本人主導の案件選択。
    - 三菱重工（MHI）、川崎重工（KHI）等の航空宇宙分野へのアプローチも「本人主導」で調整可能。
- **成長環境**: 「二十人の先輩が一人を育てる」育成体制。

## 🚫 【冷静な懐疑心】宣伝文句に隠れたリスクの再点火

リクルーターの「良い話」に酔わず、以下の冷徹な視点を意思決定に組み込みます。

### 1. 「姫路（地元）」という餌の賞味期限
- **疑問**: プロジェクト終了後も姫路でいられる保証はあるか？
- **盲点**: 「現在は案件がある」だけで、数年後の契約更新時に地元案件が消滅すれば、大手並みの全国転勤が待っている。
- **対策**: **「姫路支店の主要クライアント名とその継続年数」**を密かに調査せよ。

### 2. 「寄り添う文化」の経済的限界
- **疑問**: 不況時に会社が赤字になっても寄り添えるか？
- **盲点**: 業界構造上、エンジニアの稼働率（配属率）が90％を切れば、会社は強制的に「どこでもいいから働け」と豹変せざるを得ない。
- **対策**: 会社の「内部留保」と、リーマンショック時やコロナショック時の「具体的な待機者数」を確認する。

### 3. 「900万円」モデルの再現性
- **疑問**: あなたの技術スタック（AI/自律システム）で本当に到達可能か？
- **盲点**: その数字は「従来型の上流設計者」モデルであり、あなたの特殊な技術（Antigravity等）をどう単価に乗せるかについては実績がない。
- **対策**: 入社時に**「自分の技術スタックがどの評価テーブルに乗り、年いくらの昇給が見込めるか」**のプロトコルを文書化させる。

### 4. 競合比較の偏り
- **疑問**: メイテックやアルプスを「不当に」低く評価していないか？
- **盲点**: 他社を「悪い」と言うことで、ブレクサの「普通」が「非常に良い」に見えている可能性がある（対比効果）。メイテックが持つ「圧倒的な業界ネットワークと技術力」を捨てて、本当にブレクサの「柔軟性」を取る価値があるか。
