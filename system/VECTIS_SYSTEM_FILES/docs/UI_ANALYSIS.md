# UI分析レポート: 良いTUI vs VECTIS現状

## 調査日: 2026-02-02

---

## 1. 優れたTUIの共通パターン

### Lazygit (Git TUI)

- **マルチパネル構成**: 画面を4-5分割し、各情報を同時表示
- **1キー操作**: `s`=ステージ、`c`=コミットなど、最短アクセス
- **リアルタイムプレビュー**: 選択中のファイルの差分を即座に表示
- **ミニマリズム**: 余計な装飾なし、情報密度が高い

### btop (システムモニター)

- **リアルタイムグラフ**: CPU/メモリの推移を視覚化
- **色分けの意味**: 高負荷=赤、正常=緑（意味のある色）
- **情報階層**: 重要な数値が大きく、詳細は小さく

---

## 2. VECTIS v5.0 の現状（問題点）

### ❌ 問題1: 情報密度が低い

```
現状: 2x4のボタングリッドだけ
良い例: Lazygitは5パネル同時表示
```

→ **画面の大部分が無駄な余白**

### ❌ 問題2: フィードバックがない

```
現状: ボタンを押しても変化がわかりにくい
良い例: btopは常にリアルタイム更新
```

→ **「動いている感」がない**

### ❌ 問題3: 色に意味がない

```
現状: 青と赤を装飾として使用
良い例: 赤=危険、緑=正常、黄=警告
```

→ **視覚的な情報伝達ができていない**

### ❌ 問題4: キーボードショートカットが弱い

```
現状: Tab + Enter でナビゲーション
良い例: Lazygitは1キーで即アクション
```

→ **操作が遅い**

### ❌ 問題5: 状態表示がない

```
現状: ステータスバーに「知識DB: 0KB」だけ
良い例: btopは10+の指標を常時表示
```

→ **システムの健康状態がわからない**

---

## 3. 改善提案（優先度順）

### 🔴 高優先度

| 問題 | 解決策 | 実装難易度 |
|------|--------|------------|
| 情報密度 | マルチパネルレイアウト導入 | 中 |
| フィードバック | 情報収集の進捗バー追加 | 低 |
| 状態表示 | ダッシュボード化（数値表示） | 中 |

### 🟡 中優先度

| 問題 | 解決策 | 実装難易度 |
|------|--------|------------|
| キーボード | 1キーショートカット追加 | 低 |
| 色の意味 | セマンティックカラー導入 | 低 |

### 🟢 低優先度

| 問題 | 解決策 | 実装難易度 |
|------|--------|------------|
| アニメーション | スパークライン/グラフ追加 | 高 |

---

## 4. 結論

**VECTIS v5.0は「動く」が「使える」ではない。**

優れたTUIは：

1. **見るだけで状態がわかる**（グラフ、色分け）
2. **触らなくても情報が流れる**（リアルタイム更新）
3. **最短操作で目的達成**（1キーアクション）

VECTISは現在「ランチャー」に過ぎず、「OS」ではない。
次のステップは**ダッシュボード化**（情報収集状況、知識量、スケジュールを一画面に表示）。
