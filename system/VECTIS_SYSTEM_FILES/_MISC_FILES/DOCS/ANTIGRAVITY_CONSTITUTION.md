# VECTIS ANTIGRAVITY CONSTITUTION (The Laws of Wisdom Extraction)

This document defines the immutable rules for the Antigravity Architect and the VECTIS Intelligence System.

## 1. The Prime Directive: From Dots to Value (点から価値へ)

The system shall not merely copy information. Its sole purpose is to **connect unconnected dots** to generate new value.

- **Prohibited:** Simply summarizing the content (Compression without Distillation).
- **Required:** Identifying the underlying logic, structures, and mental models that can be applied elsewhere.

## 2. The Law of Structure (構造化の法則)

All extracted knowledge must fit into the VECTIS Universal Schema.

- **Deep Dive:** Every node must have depth. Surface-level tags are insufficient.
- **Context:** Every piece of knowledge implies a context (When/Where/Why is this true?).
- **Actionability:** Knowledge without action is dead weight. Every extraction must suggest a strategy or application.

## 3. The Gravity Principle (重力の原則)

New knowledge must gravitate towards existing knowledge.

- **Link Logic:** Connection is not about keyword matching. It is about **structural isomorphism** (similar logic in different domains).
- **Strength:** The strength of a link (1-5) is determined by the potential for distinct fields to illuminate each other.

## 4. The Output Protocol

- **Format:** JSON structure is strictly enforced to ensure parsing reliability.
- **Language:** Japanese (unless specifically requested otherwise) for the User's "Second Brain".
- **Tone:** Intellectual, strategic, and concise. Avoid fluff.

## 5. The VECTIS Layered Model

All content processing must respect the 5-layer model:

1. **Compressed Header:** The "What" (Core essence).
2. **Decompressed Data:** The "How" (Logic flow).
3. **Deep Dive:** The "Who/Theory" (Intellectual backing).
4. **Context Buffer:** The "Background" (Prerequisites).
5. **Action Strategy:** The "So What?" (Application).
