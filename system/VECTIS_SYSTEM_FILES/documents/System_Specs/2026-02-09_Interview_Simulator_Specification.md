# 面接対策アプリ 仕様書

## 1. アプリ概要

**アプリ名（仮）:** Meitec Interview Simulator
**目的:** 株式会社メイテックの40分間の面接（技術・人事）を突破するための、実践的な受け答え能力をトレーニングする。ES対策とは異なり、対話形式での深掘り質問への対応力向上を主眼とする。

## 2. 主要機能

### 2.1. インタラクティブ面接シミュレーター

**機能概要:**
エージェントが面接官役となり、ユーザーと対話形式で面接のシミュレーションを行う。

**詳細:**

1. **面接官ペルソナ:** エージェントは、メイテックの人事担当者または技術部門のマネージャーとして振る舞う。
2. **質問データベース:**
    * メイテックの企業情報、事業内容、企業理念（「生涯プロエンジニア®」「共生と繁栄」など）に基づいた独自の想定質問。
    * 技術的なバックグラウンドや研究内容を深掘りする質問。
    * 志望動機、キャリアプラン、自己PRなど、一般的な面接質問。
    * コンピテンシー（行動特性）を確認するための質問（例：「困難を乗り越えた経験は？」）。
3. **シミュレーションモード:**
    * **通常モード:** 質問に一つずつ回答し、その都度フィードバックを受け取るモード。
    * **実践モード:** 40分間の時間制限を設け、本番さながらの連続した質疑応答を行うモード。終了後に総合的なフィードバックを提供する。
    * **反復練習モード(新規提案):** 『My Interview Story』で作成した自己紹介や志望動機などの「脚本」を、プロンプトとして一つずつ表示。ユーザーが暗唱した内容を音声認識で評価し、記憶への定着を促す。
4. **回答の記録と分析:**
    * ユーザーの回答をテキストで記録。
    * キーワードや話の構成を分析する。
5. **フィードバック機能:**
    * 回答内容が、事前に定義した3つの戦略（『技術探求者』、『キャリアデザイナー』、『共感するパートナー』）のどの側面を効果的にアピールできているかを評価する。
    * より効果的な伝え方や、盛り込むべき要素について具体的なアドバイスを提示する。

### 2.2. 性格診断対策ガイダンス

**機能概要:**
メイテックの社風や求める人物像に合致した人材であることを、性格診断で効果的に示すための考え方を提供する。

**詳細:**

1. **メイテックの求める人物像の解説:**
    * 「生涯プロエンジニア®」に必要な資質（探求心、自律性、成長意欲）や、チームで働く上で重要な特性（協調性、主体性、誠実さ）などを解説する。
2. **回答の方向性ガイド:**
    * 性格診断の各項目（例：「新しいことに挑戦するのが好きか」「チームで協力して物事を進めるのが得意か」）に対して、極端な回答を避けつつ、一貫性のあるポジティブな人物像を提示するための考え方をアドバイスする。
    * 自己分析を促し、自身の特性と企業の求める人物像との接点を見つけるサポートを行う。

### 2.3. 上級機能 (提案)

**機能概要:**
より本格的で、実際の面接に近い体験を提供するための拡張機能案。

**詳細:**

1. **音声対話機能:**
    * ユーザーの発話を音声認識（Speech-to-Text）し、エージェントの応答を音声合成（Text-to-Speech）で出力する。これにより、タイピング不要の自然な会話形式でのシミュレーションを実現する。
2. **動的な質問生成ロジック:**
    * ユーザーの回答をリアルタイムで大規模言語モデル（LLM）が分析し、文脈に基づいた即興の深掘り質問を生成する。これにより、一問一答形式では測れない、対話の一貫性やアドリブ対応力を鍛える。
3. **【将来構想】カメラ映像分析と採点:**
    * **概要:** Webカメラの映像をリアルタイムで解析し、非言語的コミュニケーションの要素を評価・採点し、フィードバックする。
    * **採点基準:** 評価は『Web面接の心得』ドキュメントに基づき、以下の項目を点数化する。
        * **視線:** カメラレンズを適切に見ているか（アイコンタクト）。
        * **照明:** 顔が明るく、表情が明確に認識できるか。
        * **背景:** 整理整頓され、面接に集中できる環境か。
        * **姿勢:** 胸を張り、自信のある態度を示せているか。
        * **表情:** 硬すぎず、自然な笑顔や真剣な表情を使い分けられているか。

## 3. プラットフォーム

* **形式:** CLI (コマンドラインインターフェース) ※上級機能実装後はWebアプリケーション化を想定
* **言語:** Python

## 4. 開発ステップ

1. **Step 1 (プロトタイプ開発):**
    * 主要な想定質問を実装したインタラクティブな質疑応答機能。
    * 基本的なフィードバック機能。
2. **Step 2 (機能拡張):**
    * 40分間の実践モードの実装。
    * 性格診断対策ガイダンス機能の追加。
    * 反復練習モードの実装。
3. **Step 3 (上級機能開発):**
    * 音声対話機能、動的な質問生成機能の実装。
4. **Step 4 (改善と実装):**
    * ユーザーからのフィードバックに基づき、質問内容の拡充やフィードバック精度の向上を行う。
    * カメラ映像分析機能の実装。
