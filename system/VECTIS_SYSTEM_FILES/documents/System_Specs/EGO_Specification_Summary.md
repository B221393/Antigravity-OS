# EGO プロジェクト仕様サマリー (2026-02-04版)

> 2つの仕様書 (`EGO_SPECIFICATIONS_JP.md`, `SYSTEM_SPEC.md`) を基に、プロジェクトの全体像と技術要素を整理。

---

## 1. プロジェクト概要

- **正式名称**: EGO (Virtual Electronic Cybernetic Intelligence System)
- **コンセプト**: ユーザーの「拡張脳」として機能する、統合型の自律思考OS。
- **設計思想**:
  - **ローカルファースト**: 処理とデータを可能な限り端末内で完結させる。
  - **自律思考 (Agentic)**: 情報収集や学習をバックグラウンドで自律的に実行する。
  - **Zen & Professional**: Apple製品のような、洗練されたプロフェッショナルなUI/UX。

---

## 2. 技術要素の分類

ご要望に基づき、ハードウェアとソフトウェアの関連要素を以下に分類します。

### 2.1 ハードウェア関連要素

EGOは特定の物理デバイスに依存する設計にはなっていませんが、以下の点でハードウェアと関連します。

- **パフォーマンス要件 (メモリ)**:
  - ローカルで大規模言語モデル（LLM）を動作させるため、システムのパフォーマンスは物理的な**RAM容量**に依存します。
  - 仕様上、RAMが4GB以下の環境では、パフォーマンスを優先する軽量なECOモードに自動で切り替わります。
- **仮想物理パラメータの管理**:
  - `Auto Drive AI` の学習機能において、シミュレーション内で動作する車両の**物理的な挙動（速度、加速度、回転速度など）**をパラメータとしてJSONファイル (`settings.json`) で管理・調整しています。これは「仮想的なハードウェアスペック」の定義に相当します。

### 2.2 ソフトウェア関連要素

プロジェクトの中核はソフトウェアであり、多岐にわたる技術で構成されています。

- **システムアーキテクチャ**:
  - **3層構造**:
        1. **フロントエンド (UI層)**: `EGO_OS.html` を中心とするWebベースのダッシュボード。
        2. **バックエンド (API層)**: `system_bridge.py` (Python/Flask) が、UIと内部機能の橋渡しを行う。
        3. **バックグラウンド (自律処理層)**: 情報収集やAI学習など、重い処理を独立して実行するPythonプロセス群。

- **使用技術スタック**:
  - **フロントエンド**: HTML5, CSS3, JavaScript (ES6+, Vanilla)
  - **バックエンド**: Python 3.x, Flask (Web APIサーバー)
  - **AI / 機械学習**:
    - **大規模言語モデル(LLM)**: `Ollama` を通じてローカルで `glm-4.7-flash` を利用。文章要約、タスク解析などを行う。
    - **ニューラルネットワーク**: `PyTorch` を用いて自動運転AIのモデルを構築・学習。
    - **学習アルゴリズム**: `遺伝的アルゴリズム` を採用し、効率的に最適な運転モデルを探索。
  - **データ形式**: JSON (設定、データ交換), Markdown (レポート)

- **主要なソフトウェアコンポーネント**:
  - **EGO OS Dashboard**: 全てのアプリや情報を集約・表示するメイン画面。時間帯でモードが切り替わる。
  - **AI Assistant**: ユーザーからの自然言語指示を解釈し、適切なアプリを起動するチャットUI。
  - **System Bridge**: フロントエンドとバックエンドの機能を安全に接続するAPIサーバー。
  - **Intelligence Patrol**: YouTube, ニュースサイト, 書籍ランキングなどから情報を自動収集・分析・蓄積する自律エージェント。
  - **Auto Drive AI**: 2D/3Dシミュレータ上で、AIに運転を学習させるためのトレーニングシステム。
  - **拡張AIスキル群**: `Deep Researcher`（企業分析）、`ES Architect`（ES代筆）など、特定の目的のために後から追加可能なモジュール群。
