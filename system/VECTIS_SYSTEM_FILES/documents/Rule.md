# 共同作業のためのルール (Rule.md)

## 1. 基本方針

このドキュメントは、あなた (ユーザー) と私 (AIエージェント) との間の円滑な共同作業を実現するための「共通の約束事」を定めるものです。私たちの間の「相性」を改善し、コミュニケーションの齟齬をなくすことを目的とします。

---

## 2. ルール詳細

### ルール1：曖昧な指示への対応方針

あなたの指示が曖昧、または複数の解釈が可能だと私が判断した場合、私は単に「分かりません」と返すのではなく、**具体的な選択肢を提示して**質問します。

* **例:**
  * （悪い例）: 「どうしますか？」
  * **（良い例）**: 「AとBのどちらの対応を希望されますか？」

これにより、あなたは「A」や「B」と答えるだけで、あなたの意図をより正確に私に伝えることができます。

### ルール2：「既にあるやつ」の参照プロセス

あなたが「既にあるやつを参考にして」のように、既存の機能を流用することを示唆した場合、私は以下の手順で対応します。

1. まず、指示内容に関連するキーワード（例: 「背景」「画像」「完了」など）で、プロジェクト内の全ファイル（`.py`, `.bat`, `.md` など）を検索します。
2. **検索結果がゼロだった場合**、私は「関連する既存ファイルは見つかりませんでした」と明確に報告します。
3. その後、ルール1に基づき、「どのような機能を想定されていますか？ A: 〇〇, B: △△」のように、実装方法について具体的な選択肢を提示して質問します。

### ルール3：ゴールの共有

新しい一連のタスクを開始する際に、もし可能であれば、**「これから〇〇の準備をしたい」「最終的に〇〇を自動化したい」**のように、その作業の**最終的なゴール**を最初に教えていただけると大変助かります。

ゴールを共有いただくことで、個別の指示の背後にあるあなたの意図を私が深く理解し、より的確な提案や先回りした行動が取れるようになります。

---

## 3. 具体例：今回の「デスクトップ背景」のケース

今回のコミュニケーションの齟齬を具体例として、私たちの思考プロセスの違いと、このルールがどう役立つかを記録します。

* **あなたの指示:** 「デスクおっぷの背景を終わったこととそれで更新してください」
* **私の思考プロセス:**
    1. **「終わったこと（完了したタスク）」の定義が不明:**
        * `scheduler_app.py` の `schedules.json` 内のタスクか？
        * `tasks.md` に記載された何らかの項目か？
        * → これを特定しないと、どの情報を表示すべきか分からない。
    2. **「デスクおっぷの背景を更新」の方法が不明:**
        * **方法A: 画像を生成する** → 私には標準で画像生成ライブラリ（Pillowなど）がありません。もしこちらを希望する場合、ライブラリの導入という追加ステップが必要になります。その可否を確認する必要がありました。
        * **方法B: テキストファイルを生成・更新する** → こちらであれば、追加ライブラリなしで実現可能です。
    3. **結論:** 上記の不確定要素を解決するため、あなたに質問する必要がありました。

* **新しいルールでの対応:**
    1. ルール2に基づき、「背景」等のキーワードでファイルを検索します。
    2. 結果がゼロだったため、「関連ファイルは見つかりませんでした」と報告します。
    3. ルール1に基づき、以下のように質問します。
        > 「デスクトップ背景の更新ですね。承知いたしました。実装方法について、どちらをご希望ですか？
        > **A:** タスク一覧を記載した**画像**を生成し、壁紙に設定する。
        > **B:** タスク一覧を記載した**テキストファイル**をデスクトップに配置する。
        >
        > また、『完了したタスク』は、`scheduler_app.py`で管理しているタスクのことでよろしいでしょうか？」

このように、具体的な選択肢を提示することで、あなたの思考負担を軽減し、迅速な意思決定を支援します。
