## 1. 超高速・軽量キーワード検索ライブラリ「BM25s」

**カテゴリ**: 検索エンジン / RAG / Python  
**導入**: `pip install bm25s`

### 概要

BM25（Best Match 25）はTF-IDF系の古典的キーワード検索アルゴリズム。`bm25s` はそれを**Pythonで極限まで高速化・軽量化**したライブラリ。

### 主な特徴

- **速度**: 数百万件のドキュメントを **0.01秒以下**でインデックス検索可能。
- **省メモリ**: `mmap`（メモリマップドI/O）対応で、インデックスをメモリに丸ごと読み込まずにディスクから直接参照できる。
- **シンプルAPI**: インデックス構築と検索を数行で記述可能。

### RAGへの応用（ハイブリッド検索）

生成AI（LLM）の**RAG（Retrieval-Augmented Generation）**パイプラインでは、意味的類似性を捉える**ベクトル検索**と、キーワードの完全一致を得意とする**キーワード検索（BM25）**を組み合わせる**ハイブリッド検索**が精度向上の定石になっている。

```
[クエリ]
  ├─ ベクトル検索 (embedding + cosine similarity)
  │     └─ 「意味が近い」文書を取得
  └─ BM25s キーワード検索
        └─ 「言葉が合致する」文書を取得
              ↓
         RRF (Reciprocal Rank Fusion) で統合
              ↓
         LLMへ入力 → 回答生成
```

`bm25s` はハイブリッド検索の**キーワード検索側**を担う部品として非常に優秀。Vector DBを使わずとも高速なRAGシステムを構築可能。

### VECTIS/EGO への応用可能性

- 蓄積された企業研究ドキュメント（Knowledge_Base）の全文検索エンジンとして活用できる。
- `faiss` や `sentence-transformers` と組み合わせてローカルRAGを構築する際のキーワード検索レイヤーとして最適。

## 2. コードエディタ「Zed」の学生・教員向け無料プラン（Zed for Students）

**カテゴリ**: 開発環境 / エディタ  
**URL**: [zed.dev](https://zed.dev)

### 概要

RustとGPUを活用した**超高速コードエディタ「Zed」**が、大学生および教員向けに **「Zed Pro」プランを1年間無料**で提供開始（Zed for Students）。

### 含まれる機能

| 機能 | 内容 |
| :--- | :--- |
| **AIトークンクレジット** | 月額$10相当。Claude / GPT等のAPIをZed内から利用可能 |
| **Edit Predictions（編集予測）** | コード補完を超えた「次の編集箇所と内容」の予測・提案（無制限） |
| **高速レンダリング** | GPUアクセラレーションによるVSCode比で圧倒的なUI応答速度 |

### VSCodeとの違い

- Electron（JavaScript）ベースのVSCodeに対し、Zedは**Rust + GPU**で動作するためCPU・メモリ使用量が大幅に低い。
- Language Server Protocol (LSP) 対応で、RustやPythonのリアルタイム解析が高速。

### 活用メモ

- **学生なので今すぐ無料で申請可能**。GitHubの学生認証（Student Developer Pack）と連携。
- AI支援コーディング環境として、Copilot（VSCode）に対するオルタナティブとして評価に値する。

## 3. AIエージェント集合知シミュレーター「MiroFish」

**カテゴリ**: AI / リサーチツール / 未来予測

### 概要

大量の**AIエージェント（仮想人物）を生成し、特定の資料・テーマをもとに討論・シミュレートさせる**ことで「未来を先読み」するリサーチツール。

### 動作原理

```
[資料インプット]（業界レポート、ニュース、有価証券報告書 etc.）
       ↓
[多様なエージェント生成]（楽観派・悲観派・現場担当者・経営者・顧客 etc.）
       ↓
[仮想討論・ロールプレイ]
       ↓
[多様な意見・シナリオの抽出]
       ↓
[未来予測・戦略立案へ活用]
```

### デモ: 「SIerの未来」シミュレーション

ポストでは**SIer（システムインテグレーター）業界の未来**をMiroFishでシミュレートした結果が示唆されており、AIによる業務自動化がSIerのビジネスモデルを変容させるシナリオが示された。

### 応用可能性

- **就活への活用**: 志望企業の「10年後の姿」を多角的にシミュレートし、企業選択や志望動機の深掘りに活用。
- **コンサル的思考の訓練**: 「経営者・競合・顧客・従業員」など複数視点でシナリオを考える訓練になる。
- 経営者・コンサルタント・ストラテジストにおすすめとされている。

## 4. ノードベース・シェーダープログラミング環境（phi16氏の試作）

**カテゴリ**: グラフィック / シェーダー / ノードUI / 実験的ツール

### 概要

Xユーザー **phi16氏**によるポストで、**独自開発のノードベースUIとコードを組み合わせた環境**で**マンデルブロ集合（Mandelbrot Set）** を描画しているデモ。

### 技術的なポイント

- **ノードベースUI**: 数式やデータフローを「ノード（箱）とエッジ（接続線）」で視覚的に表現。ShaderToyやBlenderの「Geometry Nodes」に近いアプローチ。
- **シェーダー（Shader）**: GPUで並列計算を行うプログラム。マンデルブロ集合のような複雑なフラクタル図形をリアルタイムで描画できる。
- **コードとUIの統合**: ノードでロジックを組みつつ、コードで細かい制御も可能という「ハイブリッドな表現環境」。

### マンデルブロ集合とは

複素平面上の点 $c$ に対して $z_{n+1} = z_n^2 + c$（$z_0=0$）の反復が発散しないか判定した結果を色で可視化したフラクタル図形。計算がシンプルな割に無限に複雑な構造を持つことから、**並列計算（シェーダー）の定番ベンチマーク・デモ**として世界中で使われる。

### phi16氏のコメント

> 「これができればなんでもできるやろみたいな気持ちある」

複雑な数値計算とグラフィックを統合できる環境ができれば、シミュレーション・可視化・インタラクティブアート何でも応用できるという意味。

## 5. freee が MCP（Model Context Protocol）に対応

**カテゴリ**: SaaS / API / AI統合 / MCP

### 概要

クラウド会計・HR SaaSの**freee**が、GitHub上で [`freee-mcp`](https://github.com/freee/freee-mcp) を公開。**MCP（Model Context Protocol）** の実装により、ClaudeなどのLLMからfreeeのデータに直接アクセス・操作できる環境が整いつつある。

### MCPとは

**Anthropicが策定したオープン標準プロトコル**。AIエージェント（LLM）が外部ツール・APIを呼び出す際のインターフェースを統一化する仕組み。

```
[Claude / GPT etc.]
     ↓ MCPリクエスト
[MCPサーバー (freee-mcp)]
     ↓ freee API呼び出し
[freee（会計データ・経費・請求書 etc.）]
     ↓ データ返却
[LLMが回答・処理を実行]
```

### freee-mcp の構成

- `freee-api-skill` ディレクトリが確認されており、freeeの各機能（仕訳、経費精算、請求書など）がスキルとして定義されている。
- Claudeのような対話型AIから「先月の経費を集計して」「未払い請求書を一覧にして」といった自然言語指示でfreeeを操作できる未来が近い。

### 就活・業界分析視点

- freeeは **MCP対応で「AIネイティブな会計SaaS」** のポジションを早期に確立しようとしている。
- 競合（マネーフォワード・弥生）がMCP対応するかどうかが業界の注目点。
- **freeeを志望企業として研究する際の重要なプロダクト戦略の証拠**。[freee企業研究](../Company_Research/2026-02-03_freee_Research.md)と合わせて参照。

## 6. コード依存関係可視化ツール「CodeGraphContext」

**カテゴリ**: 開発ツール / コード解析 / AI補助

### 概要

コードの**「どの関数がどこを呼び出しているか（Call Graph）」をグラフとして視覚化**してくれる**無料・オープンソース**のツール。ブラウザ上で動くPlaygroundが用意されており、すぐに試せる。

### 何ができるか

```
main.py
  └─ process_data()
        ├─ load_csv()     ← utils.py
        ├─ clean_text()   ← preprocessing.py
        └─ save_result()  ← db.py
                              └─ connect()  ← config.py
```

このような**関数呼び出しの連鎖**を自動でグラフ化する。大規模コードベースで「この関数がどこから呼ばれているか」「この変更がどこに影響するか」を即座に把握できる。

### 二重の価値

| ユーザー | メリット |
| :--- | :--- |
| **人間のエンジニア** | コードリーディング時間を劇的削減。新規参画プロジェクトでの立ち上がりが早い。 |
| **AI (LLM)** | グラフ化された構造をコンテキストとして与えることで、AIがコードベース全体を把握しやすくなる。Copilotやカスタムエージェントとの連携に有効。 |

### VECTIS/EGO への応用可能性

- EGOシステム自体のコード依存関係を可視化し、リファクタリング計画の策定に活用できる。
- AIエージェント（Claude等）にコードグラフを渡すことで、「EGOのコードを理解した上での改善提案」の質が向上する。
